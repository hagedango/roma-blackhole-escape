## 0. プロジェクト概要

| 項目 | 内容 |
|---|---|
| タイトル | ロマ子様のブラックホール・エスケープ |
| ジャンル | 固定画面・引力回避アクションゲーム |
| プラットフォーム | ブラウザ（PC / スマホ両対応） |
| 技術構成 | HTML + CSS + JavaScript（Vanilla JS、外部ライブラリ・ビルドツール・音声ファイル不要） |
| 配布形態 | `index.html` をブラウザで開くだけでそのまま遊べること（サーバー不要、npm install 不要） |
| 想定プレイ時間 | 1プレイ 30秒〜2分程度のショートゲーム |
| ターゲットユーザー | ロマ子様コミュニティのメンバー（非エンジニアも含む一般ユーザー） |

### ゲームコンセプト（一言で）
画面中央付近に出現したブラックホールが、時間経過とともに**プレイヤーを引き寄せながら拡大**していく。プレイヤー（ロマ子様）は飛んでくる隕石（障害物）を避けつつ、ブラックホールの引力に逆らって**できるだけ長く生き延びる**。スコアは「生存時間＋スターのかけら収集ボーナス」の合計。

### ゲームデザインの狙い（実装者向けメモ）
- ただ避けるだけの受け身ゲームにしないため、**スターのかけら（収集アイテム）をブラックホール付近に出やすく**配置し、「安全に逃げるか、危険を冒して稼ぐか」のリスク＆リターンを作る。
- **ダッシュ（緊急回避）**により「吸い込まれそう→脱出！」の爽快感を作る。
- **ロマ子様のセリフ演出**でキャラクターゲームとしての愛嬌を出す。
- **吸い込みモード（引力サージ）**を周期的に発生させ、緊張と緩和のリズムを作る。
- **難易度曲線（v2）**：時間経過とともに隕石が速く・多くなり、最後まで緊張感が持続する。
- **罵倒/ご褒美ランク（v2）**：ゲームオーバー時にスコア別の称号とロマ子様のメッセージを表示し、スクショを見せ合いたくなる「コンテスト映え」を狙う。

## 1. 画面フローと遷移

3つの画面を持ち、すべて1つの `index.html` 内で `display: none/block` を切り替える「シングルページ・ステートマシン」方式で実装する（ページ遷移やルーティングライブラリは使わない）。

```
[タイトル画面]
   │ 難易度を選択（EASY / HARD のトグルボタンで選択状態を視覚的に表示）
   │ 「スタート」ボタンをクリック/タップ → BGM再生開始
   ▼
[プレイ画面]
   │ プレイヤーがブラックホールに吸い込まれる or 隕石に衝突
   │ →（隕石なら火花エフェクト）→ 吸い込まれ演出（約1.5秒）
   ▼
[ゲームオーバー画面]
   │ スコア別の称号＋罵倒/ご褒美メッセージを表示
   │ 「もう一度遊ぶ」→ プレイ画面へ（同じ難易度で即リトライ）
   │ 「タイトルへ戻る」→ タイトル画面へ
   ▼
```

共通UI：画面右上に**ミュートボタン（🔊/🔇）**を常時表示。ON/OFF状態は `localStorage`（キー `romako_bh_muted`）に保存し、次回起動時も引き継ぐ。

### 1-1. タイトル画面（`#screen-title`）
- 全体を**ガラスモルフィズムパネル**（3-1節）で包む
- ゲームタイトルロゴ：ピンク→紫のグラデーション文字＋ゆっくり明滅するグロー（CSSアニメーション）
- 難易度選択UI：「かんたん（EASY）」「むずかしい（HARD）」の2択ボタン。選択中はピンクの発光ボーダー＋軽い拡大。初期選択は EASY
- 操作説明（2〜3行）。「時間が経つほど隕石は速く、多くなる」ことも予告する
- 「スタート」ボタン（ホバーで浮き上がるバネ系イージング付き）
- ハイスコア表示（難易度ごとに別々に保存・表示）

### 1-2. プレイ画面（`#screen-play`）
- `<canvas>` 要素（800x600。CSSで縦横比を保ったままレスポンシブ縮小）
- canvas上部に**ガラスバーのHUD**をオーバーレイ表示：
  - 合計スコア（リアルタイム、2-9節）
  - 経過秒数
  - ダッシュクールタイムゲージ（回復完了時は発光パルス）
  - 狭い画面（520px以下）では経過秒数を隠しゲージを縮めて、ミュートボタンと重ならないようにする
- canvas内に描画するもの：
  - プレイヤー（ロマ子様。プレースホルダーはピンクの円＋簡易フェイス。3章参照）
  - ブラックホール（黒円＋紫グロー＋回転する降着円盤ふうリング）
  - 隕石／スターのかけら／セリフ吹き出し
  - パーティクル（火花・キラキラ・残像。3-2節）
  - 浮き上がるスコアテキスト（かけら取得時の「+5」等）
- スマホ対応：画面下部に十字方向ボタン＋DASH専用ボタン（`pointer: coarse` のときのみ表示）

### 1-3. ゲームオーバー画面（`#screen-gameover`）
- 画面切り替えの**前に**、プレイ画面上で「吸い込まれ演出」（2-8節）を約1.5秒再生してから遷移する
- 全体をガラスモルフィズムパネルで包み、以下を上から順に表示：
  1. 「GAME OVER」（発光文字。上から降ってくるバネ系アニメーション）
  2. ロマ子様の捨てゼリフ（2-7節のセリフ集からランダム）
  3. **称号バッジ**（2-12節のランク名。ポップインアニメーション付き）
  4. **罵倒/ご褒美メッセージ**（ランク別配列からランダム。フェードアップ表示）
  5. スコア（内訳も表示：「生存 42.3秒 ＋ かけら 8個」）
  6. New Record 表示（更新時のみ。点滅＋ファンファーレSE）
  7. ハイスコア
- 「もう一度遊ぶ」「タイトルへ戻る」ボタン

## 2. コアロジックとルール

### 2-1. 操作方法
- **PC**：矢印キー（↑↓←→）または WASD で8方向移動。**スペースキーでダッシュ**（2-6節）
- **スマホ**：画面下部の十字方向ボタン（タップ＆ホールドで継続移動）＋右下のDASHボタン
- 斜め移動が直線移動より速くならないよう、入力ベクトルを正規化してから速度を掛ける

### 2-2. ブラックホールの挙動
- ゲーム開始時、canvas中央付近のランダム位置に半径 `BH_INITIAL_RADIUS` で出現
- 半径は `BH_GROWTH_RATE`（px/秒）で線形に拡大し続ける
- **引力効果**：毎フレーム、プレイヤーをブラックホール中心方向へ引っ張る
  - `force = BH_GRAVITY_STRENGTH / max(distance, 50)`（px/秒）
  - プレイヤーの最終移動 = 移動入力ベクトル + 引力ベクトル の合成
  - 近づくほど操作が効きにくくなる緊張感を演出する
- **吸い込みモード（引力サージ）**：一定間隔で短時間だけ引力が大幅に強くなるイベント
  - `SURGE_INTERVAL` 秒ごとに発動。発動の `SURGE_WARNING` 秒前から**予兆演出**：画面全体を±3px揺らし、ブラックホールの縁を赤紫に発光させ、警報SEを鳴らす
  - 予兆の後、`SURGE_DURATION` 秒間だけ引力係数を `SURGE_MULTIPLIER` 倍にする。サージ中は縁の発光を強め、周囲に回転する吸い込み線を描く
  - サージ終了で即座に通常値へ戻り、次のサージまでのカウントがリセットされる
  - 初回はゲーム開始から `SURGE_INTERVAL` 秒後（開始直後には発動しない）
  - 状態遷移は「待機(idle)→予兆(warning)→発動(active)→待機」のステートマシンで管理する

### 2-3. 隕石（障害物）の挙動
- `meteorIntervalNow()`（2-11節）の間隔で、canvas外周のランダムな位置から出現
- 出現後、canvas内側のランダムな点へ向かって `meteorSpeedNow()`（2-11節）で直進する
- 隕石もブラックホールの引力の影響を受け、軌道が中心側に曲がる（引力の0.5倍を速度に加算）
- canvas外に完全に出た隕石、ブラックホールに飲み込まれた隕石は配列から削除する（メモリリーク防止）

### 2-4. 当たり判定
- すべて円同士の衝突判定：`distance(A, B) < (radiusA + radiusB)`
- 以下のいずれかで即ゲームオーバー（→2-8節の演出へ）：
  1. プレイヤーとブラックホールの衝突
  2. プレイヤーといずれかの隕石の衝突
- プレイヤーとスターのかけらの衝突は**収集**として処理（2-5節）
- プレイヤーはcanvas境界でクランプ（画面端で止まる。ゲームオーバーにはしない）

### 2-5. スターのかけら（収集アイテム）
リスク＆リターンの核。**わざとブラックホールの近くに出やすくする**のが最重要ポイント。

- `STAR_SPAWN_INTERVAL` 秒ごとに1個出現。同時存在数は最大 `STAR_MAX_COUNT` 個
- **出現位置**：ブラックホール中心から距離 `BH半径 + 60px` 〜 `BH半径 + 200px` のドーナツ状範囲内でランダム（canvas外にはみ出す場合は再抽選、最大20回）
- かけらは静止。ブラックホールの拡大に飲み込まれたら消滅
- `STAR_LIFETIME` 秒で消える（残り2秒間は点滅）
- 収集時：`STAR_BONUS` 点を加算し、以下を同時に発火する
  - 金色のキラキラパーティクル（3-2節）
  - 浮き上がる「+N」スコアテキスト
  - 収集SE（4章）＋収集セリフ（2-7節）

### 2-6. ダッシュ（緊急回避）
- 発動：スペースキー（スマホはDASHボタン）
- 効果：**現在の移動入力方向**へ `DASH_SPEED_MULTIPLIER` 倍速で `DASH_DURATION` 秒間高速移動。ニュートラル状態で押した場合は**ブラックホールと逆方向**へダッシュ（緊急脱出として直感的）
- ダッシュ中も引力・当たり判定は有効（無敵にしない）
- クールタイム：`DASH_COOLDOWN` 秒。HUDのゲージに回復状況を表示し、満タン時は発光パルス
- 演出：発動時に青白い粒子バースト＋風切りSE＋ダッシュセリフ。移動中は半透明の残像を残す

### 2-7. ロマ子様のセリフ演出
- 表示方法：プレイヤー頭上に吹き出し（白背景・角丸＋しっぽ）を約2秒表示。新イベントで上書き
- セリフはイベント種別ごとに**配列で定義**し、ファイル冒頭の定数部にまとめる（コミュニティで差し替えやすくするため）

| イベント | 発動条件 | セリフ例（配列からランダム） |
|---|---|---|
| 危険接近 | BH表面との距離が80px未満（一度出たら5秒間は再発動しない） | 「ちょっ、引っ張らないでくださる!?」「吸わないでちょうだい！」 |
| かけら収集 | スターのかけらを取った瞬間 | 「いただきですわ✨」「当然の報酬ですわね」 |
| ダッシュ発動 | ダッシュした瞬間 | 「失礼しますわよ！」「ロマ子様、緊急回避！」 |
| サージ予兆 | 吸い込みモードの予兆演出が始まった瞬間 | 「な、なんか嫌な予感がしますわ…」「来ますわよ…！」 |
| 長生き達成 | 生存30秒・60秒到達時 | 「まだまだ余裕ですわ」「宇宙もわたくしの前ではこの程度」 |
| ゲームオーバー | 死亡演出後、ゲームオーバー画面に表示 | 「覚えてらっしゃい…！」「こ、今回は引き分けですわ…」 |

### 2-8. ゲームオーバー演出（火花＋吸い込まれアニメーション）
悔しさより笑いが先に来る演出にする。

1. **衝突の瞬間**（隕石衝突の場合のみ）：
   - 火花パーティクルを32個バースト（3-2節）
   - 爆発SE＋画面シェイク（0.35秒、振幅9px、時間とともに減衰）
2. **吸い込まれフェーズ（約1.5秒）**：
   - プレイヤーがブラックホール中心に向かって螺旋を描きながら吸い込まれる
     - 毎フレーム、BH中心からの距離を線形に縮小しつつ、角度を加算（1.5秒で2回転）
     - プレイヤーの描画サイズも徐々に縮小し、最後は消える
   - 吸い込まれながら紫〜ピンクの粒子を撒き散らす
   - 吸い込まれSE（下降グライド音）を再生
   - 演出中は操作入力・スコア加算・新規隕石の生成を停止する
3. 演出終了後、BGMをフェードアウトしてゲームオーバー画面へ遷移する

### 2-9. スコア計算
- スコア = **生存時間（秒）＋ スターのかけら収集ボーナス**
  - 生存時間：1秒＝1点（表示は小数点1桁）
  - かけらボーナス：1個につき `STAR_BONUS` 点
- HUDには合計スコアをリアルタイム表示、ゲームオーバー画面で内訳を表示
- ハイスコアは `localStorage` に難易度別キー（`romako_bh_highscore_easy` / `romako_bh_highscore_hard`）で保存

### 2-11. 難易度曲線（v2：時間経過で加速する隕石）
「後半がヌルい」を防ぎ、最後まで緊張感を持続させる仕組み。隕石の**速度**と**出現頻度**が経過時間 `t`（秒）とともに上がる。

```
共通定数 RAMP:
  SPEED_MAX_MULT  = 2.2   … 隕石速度の最大倍率
  SPEED_RAMP_TIME = 75    … この秒数かけて最大倍率へ（線形）
  SPAWN_RAMP_TIME = 45    … 出現間隔の短縮ペース
  SPAWN_MIN_RATIO = 0.3   … 出現間隔は初期値の30%までしか縮まない

meteorSpeedNow()    = METEOR_SPEED × (1 + min(t / SPEED_RAMP_TIME, 1) × (SPEED_MAX_MULT − 1))
meteorIntervalNow() = max(METEOR_SPAWN_INTERVAL / (1 + t / SPAWN_RAMP_TIME),
                          METEOR_SPAWN_INTERVAL × SPAWN_MIN_RATIO)
```

実測例（EASY）：速度 120 → 235（60秒）→ 264（120秒、上限）／出現間隔 1.5秒 → 0.64秒（60秒）→ 0.45秒（下限）。
速度は上限倍率で頭打ち、間隔は下限で頭打ちになるため、理不尽な無理ゲーにはならない。

### 2-12. 罵倒/ご褒美ランクシステム（v2）
ゲームオーバー時、最終スコアに応じた**称号**と**ロマ子様からのメッセージ**を表示する。低スコアは罵倒、高スコアはツンデレ〜ご褒美へ変化し、「何の称号だった？」とスクショを見せ合う文化を狙う。

判定は上から順に「スコア ≧ min」で決定する：

| min | 称号 | 系統 | メッセージ例（配列からランダム） |
|---|---|---|---|
| 150 | 銀河の覇者ブタ野郎 | ご褒美 | 「あ、あなた…本当にあのブタ野郎ですの!? み、認めますわ…あなたは伝説ですわ…！」 |
| 100 | 恒星級ブタ野郎 | ご褒美 | 「素晴らしいですわ！ご褒美に、わたくしの微笑みを差し上げますわ💖」 |
| 50 | 彗星級ブタ野郎 | ツンデレ | 「や、やりますわね…！べ、別に感心してなんかいませんことよ！」 |
| 20 | 小惑星級ブタ野郎 | 罵倒 | 「まぁ…ブタにしては、ほんの少しだけ頑張った方ですわね」 |
| 0 | 宇宙の塵ブタ野郎 | 罵倒 | 「話になりませんわ！ブラックホールに謝ってらっしゃい、このブタ野郎！」 |

- ランク定義（min・称号・メッセージ配列）はファイル冒頭の定数 `SCORE_RANKS` にまとめ、コミュニティで自由に増減・差し替えできるようにする
- 各ランクのメッセージは2種類以上用意し、ランダムで1つ表示する

## 3. プレミアム演出仕様（v2）

### 3-1. ガラスモルフィズムUI
すべてのUIパネルを「すりガラス」で統一し、宇宙背景が透けて見える高級感を出す。

- **基本レシピ（CSS）**：
  - `background: rgba(255,255,255, 0.05〜0.07)`（ごく薄い白）
  - `backdrop-filter: blur(10〜16px) saturate(140%)`（`-webkit-` プレフィックスも併記）
  - `border: 1px solid rgba(255,255,255, 0.12〜0.16)`
  - `box-shadow`：外側に深い落ち影＋`inset 0 1px 0 rgba(255,255,255,0.1)` で上辺ハイライト
  - 角丸は大きめ（パネル26px、バー・ボタンは999px）
- 適用箇所：タイトルパネル／ゲームオーバーパネル／HUDバー／ハイスコアボックス／結果ボックス／ミュートボタン／スマホ用ボタン
- ページ背景には CSSのみで流れる星空（`radial-gradient` を複数重ねた `background-image` を `background-position` アニメーションでドリフト）を敷く

### 3-2. パーティクルシステム
canvas内の演出はすべて共通のパーティクル機構で扱う。1粒 = `{x, y, vx, vy, life, maxLife, size, color, drag}`。

- 毎フレーム：位置を速度で更新 → 速度に減衰（drag）を掛ける → life を減らし、0以下で配列から除去
- 描画は `globalCompositeOperation: 'lighter'`（加算合成）＋ life 残量に応じたアルファで「光の粒」に見せる
- 発火イベントと初期値：

| イベント | 個数 | 色 | 速度 (px/秒) | 寿命 (秒) |
|---|---|---|---|---|
| 隕石衝突の火花 | 32 | 金・オレンジ・ピンク・白 | 80〜420 | 0.4〜1.0 |
| かけら収集のキラキラ | 14 | 金・淡黄・白 | 40〜220 | 0.3〜0.7 |
| ダッシュ発動バースト | 10 | 水色・青・白 | 40〜160 | 0.2〜0.5 |
| 吸い込まれ中の粒子 | 毎フレーム2個（確率70%） | 紫・ピンク・水色 | 20〜120 | 0.3〜0.7 |

- 同機構で**浮き上がりテキスト**（かけら取得時の「+5」）も管理：上方向へ44px/秒で浮きながらフェードアウト（寿命0.9秒）
- **画面シェイク**：衝突時（0.35秒・振幅9px・線形減衰）とサージ予兆/発動中（±3px）は、描画全体を `ctx.translate` でランダムオフセットする

### 3-3. CSSイージングアニメーション
- 画面切り替え：`screenIn`（フェード＋上方向スライド＋微スケール、0.55秒、`cubic-bezier(0.22, 1, 0.36, 1)`）。`display:none` → 表示のたびに再生される
- ボタンホバー/選択：バネ系 `cubic-bezier(0.34, 1.56, 0.64, 1)` で浮き上がり・拡大
- タイトルロゴ：`titleGlow`（グローの強弱を2.8秒で往復）
- GAME OVER文字：`titleDrop`（上から降ってバウンド）
- 称号バッジ：`badgePop`（0.25秒遅延でポップイン）→ メッセージ：`fadeUp`（0.45秒遅延）と時間差で見せる
- ダッシュゲージ満タン：`dashPulse`（明度パルス）／New Record：`recordBlink`（点滅＋微拡大）

## 4. サウンド仕様（Web Audio API・音源ファイル不要）

すべての音を**Web Audio APIでリアルタイム合成**する。音声ファイルを同梱しないため、配布は3ファイルのまま。

### 4-1. 基本設計
- `AudioContext` はブラウザの自動再生制限のため、**最初のユーザー操作（ボタンクリック）時に生成/resume**する
- ノード構成：`master`（全体音量0.6）─ `bgmGain`（0.16）／`seGain`（0.55）を分岐。ミュートは `master.gain = 0`
- **宇宙リバーブ**：`DelayNode`（0.34秒）＋フィードバックゲイン（0.38）のエコーループをBGM系統に接続
- ミュート状態は `localStorage`（`romako_bh_muted`）に保存

### 4-2. BGM（宇宙感ドローン＋アルペジオ）
- **ドローン**：ノコギリ波オシレーター3本（55Hz×2本デチューン＋110Hz）→ ローパスフィルタ（220Hz）→ 音量0.05。開始時2秒かけてフェードイン
- **アルペジオ**：Aマイナー系の音列 `A3 → C4 → E4 → G4 → C5 → G4 → E4 → C4` を0.62秒間隔でループ。サイン波・1音0.55秒・エコー送りで幻想的な響きにする
- スケジューリングは「先読み方式」：120msごとのタイマーで `nextNoteTime < currentTime + 0.35` の間、次の音を予約する（タイマー精度に依存しない）
- ゲーム開始で再生、ゲームオーバーで1秒かけてフェードアウト停止

### 4-3. 効果音（SE）レシピ

| SE | 発音タイミング | 合成レシピ |
|---|---|---|
| クリック | 各ボタン押下 | 三角波 660→880Hz、0.08秒 |
| かけら収集 | 収集時 | サイン波 880Hz → 1320Hz の2連音（キラッ） |
| ダッシュ | ダッシュ発動 | ホワイトノイズ＋バンドパス 2400→300Hz スイープ、0.22秒（風切り） |
| 爆発 | 隕石衝突 | ノイズ（ローパス2000→120Hz）＋ノコギリ波 180→40Hz、0.45秒 |
| 警報 | サージ予兆開始 | 矩形波 523/415/523Hz の3連アラーム |
| 吸い込まれ | 死亡演出開始 | サイン波 420→35Hz グライド1.3秒＋沈み込むノイズ |
| ファンファーレ | New Record時 | 三角波 523→659→784Hz（ドミソ）の3連音 |

## 5. ファイル構成

```
romako-blackhole-escape/
├── index.html      … 3画面分のHTML構造＋ミュートボタン＋HUD＋スマホ用コントロール
```

- ロマ子様はCanvasに描くピンクの円＋簡易フェイスのプレースホルダー。描画は `drawPlayer()` に分離してあり、後で画像に差し替える場合も1関数の変更で済む
- **すべての調整可能な値（難易度パラメータ・RAMP定数・セリフ集・ランク定義）は `game.js` の冒頭にまとめる**こと。コミュニティメンバーがコードを読めなくても差し替えられる構造が必須要件

## 7. 実装上の注意事項

- 外部ライブラリ・npm・ビルドツールは一切使用しない。`index.html` を直接開くだけで完全動作する構成にすること
- ゲームループの `dt` は上限クランプ（0.05秒）を入れ、タブ復帰時の物理暴走を防ぐこと
- パーティクル・隕石・かけらの配列は毎フレーム不要要素を除去し、長時間プレイでも肥大化しないこと
- `AudioContext` の生成・resumeは必ずユーザー操作イベント内で行うこと（自動再生制限対策）
- コメントは最小限でよいが、状態管理・衝突判定・音声スケジューリングなど複雑な箇所には簡潔な説明を残すこと
