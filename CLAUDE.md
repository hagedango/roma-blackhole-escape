# ロマ子様のブラックホール・エスケープ v2 - AI開発ガイド

このファイルは、AIエージェント（Opus, Codex, Claude Code等）が本プロジェクトを継続して安全に開発・メンテナンスするための指示・規約書です。

## 1. プロジェクト概要
- **目的**: ロマ子様コミュニティのゲームコンテスト用ミニゲーム
- **技術スタック**: HTML5 / CSS3 / JavaScript (Vanilla JS, Canvas API, Web Audio API)
- **特徴**: 外部の音声ファイルや重いフレームワークを使わず、`index.html` 単体（または少数のアセット）で軽量・高速に動作するシングルページアプリケーション。

## 2. 起動・ビルド方法
- ビルドステップはありません。ローカルの `index.html` を直接任意のモダンブラウザ（Chrome等）で開くことで動作します。
- デプロイ先: Netlify

## 3. ディレクトリ構成
- `index.html`: ゲームのメインHTML。
- `game.js`: ゲームロジック（Canvas描画、当たり判定、スコア、難易度曲線、サウンド生成等）。
- `style.css`: ゲーム画面やガラスモルフィズムのスタイル。
- `DESIGN_DOC.md`: ゲームデザイン、数式、アセット仕様をまとめた詳細設計図。
- `assets/images/`: キャラクター（ロマ子様、ブタ野郎等）の差し替え用グラフィック画像。

## 4. 開発・コーディング規約（AI用・厳守）
- **Web Audio APIによるサウンド生成**:
  - BGM（ドローン＋アルペジオ）およびSE7種（爆発音、ダッシュ風切り音等）は、すべてコード内の `AudioContext` を介したシンセサイザー処理で動的に生成されています。音声ファイルをロードする設計に変更する際は、既存の動的生成コードを壊さないよう注意してください。
- **パーティクルシステム**:
  - 隕石衝突時の火花（オレンジ/ピンク・加算合成発光）、アイテム回収時の金色、ダッシュ時の青、吸い込み時の紫の4種の演出を Canvas 上で描画しています。描画パフォーマンスを維持するため、パーティクルの最大個数と寿命を適切に管理してください。
- **難易度曲線（加速ロジック）**:
  - `meteorSpeedNow()` および `meteorIntervalNow()` による経過時間に応じた加速ロジックが組み込まれています。プレイヤーに理不尽さを感じさせないよう、必ず上限・下限値（リミッター）の定数を維持してください。
- **罵倒・ご褒美ランクシステム**:
  - スコア境界値（20点、50点、100点、150点）で、5段階の罵倒/ご褒美メッセージを表示します。メッセージ内容は配列で定義されており、コミュニティの文脈に合わせて自由に変更可能です。
- **ロマ子様（罵尻ロマ子）の口調ルール（セリフ追加・変更時は厳守）**:
  - 一人称は「ロマ子様」。ファンの呼称は個別「ブタ野郎」、集団「ブタ共」「おめぇら」「テメェら」。
  - 語尾は「〜だじょ！」「〜のだわわ！」「〜だかんな！」。相槌は「へーんだ！！」（突き放し）「んむ！！」（肯定）。
  - 「〜ですわ」等のお嬢様口調は**誤り**なので使わない。
  - 罵倒は「本当に傷つける一歩手前（ギリギリアウト）」で止める愛のあるスタイル。高評価時は「立派なマゾ豚として出荷」等の独自用語や、照れ隠し（「べ、別に褒めてねぇし！」）を使う。
- **ロマ子様アイコンの描画サイズ**:
  - `PLAYER_DRAW_SCALE` は見た目だけの拡大率（当たり判定は `PLAYER_RADIUS` のまま＝プレイヤー有利）。見た目を変えたい時はこの定数を触り、当たり判定を変えたい時のみ `PLAYER_RADIUS` を変更する。
